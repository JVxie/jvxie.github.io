题目大意

这是CCPC2019网络预选赛的题目，题目传送门。

给你一个数组aaa，初始是[1,n][1, n][1,n]的某个排列，有mmm个操作，操作分为两种：

  1. (1,pos)(1, pos)(1,pos)：将aposa_{pos}apos​修改为apos+10,000,000a_{pos} + 10,000,000apos​+10,000,000；
  2. (2,r,k)(2, r, k)(2,r,k)：询问一个a1,a2,……,ar{a_1, a_2, ……, a_r}a1​,a2​,……,ar​之中未出现的最小正整数AnsAnsAns且满足Ans≥kAns \geq kAns≥k。

这些给出的操作将经过加密，你需要解密，解密方式如下：

  • 每组测试数据的第一个操作前定义一个LastAnsLastAnsLastAns初始化为000；
  • 若给出两个数(1,t1)(1, t1)(1,t1)，代表这次操作是操作111，最终的pos=t1⨁LastAnspos = t1 \bigoplus LastAnspos=t1⨁LastAns；
  • 若给出三个数(2,t2,t3)(2, t2, t3)(2,t2,t3)，代表这次操作是操作222，最终的r=t2⨁LastAns,k=t3⨁LastAnsr = t2 \bigoplus LastAns, k = t3 \bigoplus LastAnsr=t2⨁LastAns,k=t3⨁LastAns；
  • 对于每个操作222得出的答案AnsAnsAns，令LastAns=AnsLastAns = AnsLastAns=Ans；
  • 其中⨁\bigoplus⨁代表的是逻辑运算中的XORXORXOR（喜闻乐见的按位异或）。

数据范围：

有T(1≤T≤10)T(1 \leq T \leq 10)T(1≤T≤10)组测试数据，保证每组测试数据给出的n(1≤n≤100,000)n(1 \leq n \leq 100,000)n(1≤n≤100,000)，m(1≤m≤100,000)m(1 \leq m \leq 100,000)m(1≤m≤100,000)，经解密过的pos,r,k∈[1,n]pos, r, k \in [1, n]pos,r,k∈[1,n]。对于所有的测试数据保证∑n≤510,000\sum n \leq 510,000∑n≤510,000，∑m≤510,000\sum m \leq 510,000∑m≤510,000。

时间、空间限制：

时间：4000/2000 MS (Java/Others)，空间：262144/262144 K (Java/Others)

题解（思路）

这个题目是队友在最后四十分钟想出来的做法……可惜我Debug的时间太长，差2秒左右时间提交，赛后补题我第一时间提交，一遍AC。只是CCPC分站赛注定与我无瓜啦（觉得有一点点对不起队友）。

首先想到，如果操作没有经过加密，我们可以使用类似莫队的算法将操作离线，然后用一个数据结构（如STL里的set，需要修改）维护当前a1……ara_1……a_ra1​……ar​中有什么值，然后对于每个操作2，只要在[k,n+1][k, n + 1][k,n+1]中二分答案，然后找到满足条件的值即可。时间复杂度大概在O(m⋅n⋅(log⁡2n)2)O(m \cdot {\sqrt n} \cdot {(\log_2n)^2})O(m⋅n​⋅(log2​n)2)，貌似勉强可以？

此处收回上述做法，答案并不符合单调性，不能二分（逃

当然……出题人显然不想让你用离线操作，于是对操作进行加密。

观察到数组aaa是某个[1,n][1, n][1,n]的排列，且1≤n≤100,0001 \leq n \leq 100,0001≤n≤100,000，所以实际上每次操作111实际上是将aposa_{pos}apos​变成操作222可能的答案之一。再者，对于每次操作222的询问，实际上ar+1……ana_{r+1}……a_nar+1​……an​的值都是可以取的答案。如果没有过操作111，且本次操作222给出的r=nr = nr=n，由于保证原数组是个排列，不会出现重复元素，所以答案则是n+1n + 1n+1。

最终得出，先将给出的初始序列aaa丢入一个数据结构，使得其在O(log⁡2n)O(\log_2n)O(log2​n)的时间复杂度下支持查询某个数在某个区间内所在位置（显然主席树或划分树可用）。对于每次操作111，将aposa_{pos}apos​丢到一个在O(log⁡2n)O(\log_2n)O(log2​n)的时间复杂度下支持插入、查询的数据结构setsetset中（没错就是可以用STL里的set）。对于每次操作222，查询setsetset中的第一个≥k\geq k≥k的数字，再找到ar+1……ana_{r+1}……a_nar+1​……an​中第一个≥k\geq k≥k的数字，答案就是较小的那个。如果找不到？那答案自然就是n+1n + 1n+1。

以上做法时间复杂度大概在O(m⋅(log⁡2n)2)O(m \cdot {(\log_2n)^2})O(m⋅(log2​n)2)，如果是用主席树做的，可能可以压缩到O(m⋅log⁡2n)O(m \cdot {\log_2 n})O(m⋅log2​n)，但对于本题来说，时间是很充裕的啦。
